# Point: Azure DevOps Artifacts Feed Ownership

## Approved Phrasings

- Preferred phrasing:
  `Owned the Azure DevOps Artifacts npm feed for Copilot Hub shared components, managing package publishing, PR-based promotion, rollback, and CI-validated updates`
- Accepted detailed phrasing:
  `Owned the Azure DevOps Artifacts npm feed for Copilot Hub shared components, managing package publishing, PR-based promotion, rollback, and CI-validated updates between the component repository and the consuming platform repository`

## Technologies And Boxes

- Azure DevOps Artifacts
- npm package publishing
- Artifact versioning
- Release promotion
- Rollback
- CI/CD

## Allowed Headers

- `CI/CD & Internal Tooling`
- `Testing, CI/CD, & Internal Tooling`
- `Operations, Reliability, & Production Support`

## Notes

- Distinct from the broader CI/CD ownership point, but overlaps with that story and should usually replace the weaker Azure DevOps repositories point when artifact/versioning detail matters more.
