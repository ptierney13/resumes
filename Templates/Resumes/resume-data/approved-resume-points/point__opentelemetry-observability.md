# Point: OpenTelemetry Observability

## Approved Phrasings

- Preferred for DevOps and observability-first roles:
  `Co-designed and implemented an OpenTelemetry instrumentation framework for Microsoft Whiteboard, making observability consistent across the service while keeping developer overhead low`
- Preferred for platform/operations roles:
  `Co-designed and implemented a telemetry platform for Microsoft Whiteboard using OpenTelemetry, making the service easier to diagnose in production while keeping developer overhead low`
- Accepted detailed phrasing:
  `Worked with 4 engineers to design and implement an architecture via OpenTelemetry, crafting APIs that enforced detailed telemetry generation`

## Technologies And Boxes

- OpenTelemetry
- Observability
- Distributed tracing
- Platform diagnostics
- Developer experience

## Allowed Headers

- `Observability & Telemetry`
- `Platform & Framework Development`
- `Operations, Reliability, & Production Support`

## Notes

- The `telemetry platform` wording is an approved role-specific variant, not the universal default.
